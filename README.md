## 1. 产品概述

### 1.1 产品简介
智能变化检测系统是一款基于计算机视觉技术的实时监控软件，能够检测并标记屏幕或摄像头画面中的动态变化区域。本系统采用多种加速技术（包括CUDA、OpenCL、Numba和PyTorch）实现高性能处理，适用于游戏开发、安全监控、用户行为分析等多种场景。

## 2. 安装指南

### 2.1 基础安装
1. 确保已安装Python 3.6或更高版本
2. 安装依赖库：
   ```
   pip install opencv-python numpy pillow mss pywin32 psutil
   ```

### 2.2 可选加速支持
- **CUDA加速**：需安装支持CUDA的OpenCV版本
- **OpenCL加速**：需安装支持OpenCL的显卡驱动
- **Numba加速**：`pip install numba`
- **PyTorch加速**：`pip install torch`

## 3. 功能说明

### 3.1 核心功能
- **实时变化检测**：检测画面中的动态变化区域
- **多源监控**：支持监控应用程序窗口或摄像头
- **性能优化**：自动选择最佳加速后端
- **区域选择**：可自定义监控区域
- **参数调节**：灵活调整检测敏感度和帧率

### 3.2 技术特点
- 采用帧间差分算法检测变化
- 支持多种硬件加速技术
- 自适应性能调节
- 渐变动画效果显示

## 4. 使用指南

### 4.1 界面说明
![界面布局示意图]
1. **视频显示区**：实时显示监控画面和检测结果
2. **控制面板**：参数设置区域
3. **状态栏**：显示系统状态和性能信息

### 4.2 基本操作

#### 4.2.1 选择监控源
1. 点击"选择监控进程"按钮选择应用程序窗口
2. 或点击"选择镜头监控"按钮选择摄像头

#### 4.2.2 设置监控区域
1. 选择监控进程后，点击"选择监控区域"按钮
2. 在目标窗口上拖动鼠标选择区域
3. 释放鼠标左键确认选择

#### 4.2.3 参数调整
- **帧率(FPS)**：控制处理速度（1-360）
- **变化阈值**：调整检测敏感度（1-100）
- **最小区域**：过滤小面积变化（1-1000像素）

### 4.3 高级功能

#### 4.3.1 加速后端选择
在"加速设置"区域可选择：
- CUDA（NVIDIA显卡）
- OpenCL（支持OpenCL的硬件）
- Numba（JIT编译加速）
- PyTorch（GPU加速）
- CPU（默认后备方案）

#### 4.3.2 显示选项
- **保持比例**：保持原始画面宽高比
- **总是置顶**：窗口保持在最前端

## 5. 技术原理

### 5.1 算法流程
1. 获取连续两帧图像
2. 转换为灰度图并计算差异
3. 二值化处理
4. 轮廓检测和过滤
5. 绘制检测结果

### 5.2 加速技术对比

| 技术 | 适用场景 | 优势 | 限制 |
|------|---------|------|------|
| CUDA | NVIDIA显卡 | 高性能并行计算 | 需专用硬件 |
| OpenCL | 多平台GPU | 跨平台支持 | 性能依赖驱动 |
| Numba | CPU加速 | 简单易用 | 首次编译耗时 |
| PyTorch | 深度学习 | 灵活性强 | 内存占用高 |
| CPU | 通用 | 无需特殊硬件 | 性能有限 |

## 7. 常见问题

### Q1: 检测结果不准确怎么办？
A: 尝试调整"变化阈值"和"最小区域"参数，或选择更合适的监控区域。

### Q2: 性能不足怎么办？
A: 1) 降低帧率设置 2) 选择更适合的加速后端 3) 缩小监控区域

### Q3: 如何选择最佳加速后端？
A: 系统会自动检测，也可在"加速设置"中手动测试不同选项。
